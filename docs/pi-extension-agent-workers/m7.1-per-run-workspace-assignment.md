# M7.1 — Per-run workspace assignment

## Status

Done.

## SPEC

### Scope

Adjust M7 workspace behavior so worker workspace selection is per-run, not sticky session state.

Manual acceptance showed that a persistent selected workspace does not match the intended workflow:

1. In the simple case, the worker should just run in the current pi cwd.
2. In a central orchestration repo, different workers may be assigned to different product repositories.
3. Hidden session fallback makes it harder to reason about where a worker will run.

M7.1 should make the effective cwd rule simple and explicit:

```text
effective cwd = explicit cwd || current pi cwd
```

### Target behavior

Commands:

- `/worker-workspace` shows the current cwd, git root, and preflight hints only. No selected/effective persistent state.
- `/worker-run --cwd <path> ...` runs in that explicit cwd for this run.
- `/worker-run --pick-cwd ...` opens native `ctx.ui.select()` / `ctx.ui.input()` and uses the chosen cwd for this run only.
- `/worker-run ...` without `--cwd` or `--pick-cwd` uses the current pi cwd.
- `/worker-workspace-pick` may remain as a non-sticky helper that prints a selected path/preflight result, but must not set a default.

Tools:

- `agent_worker_start({ cwd })` uses that cwd for the run.
- `agent_worker_start` without cwd uses the current tool context cwd.
- No selected workspace fallback.

### Non-goals

- No persistent workspace registry.
- No session default workspace.
- No custom TUI.
- No Jira-specific workspace mapping.
- No model calls required for validation.

## AC

Implementation is complete when:

1. There is no selected session workspace fallback.
2. `/worker-run` without cwd uses current pi cwd.
3. `/worker-run --cwd <path>` uses explicit cwd for one run.
4. `/worker-run --pick-cwd ...` uses native picker and applies the chosen cwd only to that run.
5. `/worker-workspace` no longer shows selected workspace.
6. `/worker-workspace-pick` does not persist a selected workspace.
7. `agent_worker_start` without cwd uses tool context cwd, not a hidden selected workspace.
8. Docs describe per-run workspace assignment.
9. Tests cover the changed behavior.

- Changed `/worker-workspace-pick` to print a picked path/preflight result without setting defaults.
- Updated docs to describe per-run workspace assignment.
