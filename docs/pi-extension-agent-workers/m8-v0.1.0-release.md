Done.

## SPEC

### Scope

Prepare `@gregho/pi-extension-agent-workers` for its first usable local package release as `v0.1.0`.

M0-M7.1 delivered the initial generic worker runtime:

- safe demo worker supervision
- Claude Code and Codex CLI adapters
- normalized event/usage parsing
- reusable service/profile API
- LLM-callable `agent_worker_*` tools
- orchestration recipes
- per-run workspace assignment and preflight

M8 should convert that accumulated pre-release work into a coherent `0.1.0` package version and release documentation.

- Package version is bumped from `0.0.0` to `0.1.0`.
- Changelog has a dated `0.1.0` release section summarizing M1-M7.1.
- README status describes `v0.1.0` as the current initial release.
- Docs index/current status mention `v0.1.0` readiness.
- Release verification commands pass.
- Pack dry-run tarball reports version `0.1.0`.

- `packages/pi-extension-agent-workers/package.json`

### Package-lock policy

`package-lock.json` has an existing unrelated uncommitted change in this worktree. M8 should not commit unrelated lockfile changes.

If changing `package.json` version normally requires lockfile synchronization, either:

1. update only the package-level `package.json` and leave the existing root lockfile untouched, documenting that this repo is using source-loaded pi packages; or
2. if a lockfile update is necessary, inspect the diff carefully and commit only the version-related lockfile hunks if they can be isolated safely.

Do not blindly commit the existing `package-lock.json` diff.

### Non-goals

- No new runtime behavior.
- No new worker adapters.
- No domain-specific runtime surfaces.
- No publishing to npm unless explicitly requested.
- No git tag unless explicitly requested.
- No real Claude/Codex model smoke unless explicitly approved.
